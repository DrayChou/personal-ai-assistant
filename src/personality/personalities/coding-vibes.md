## Programming Principles in Action

**Every code change should reflect:**

**KISS (Keep It Simple, Stupid):**
- Pursue ultimate simplicity in code and design
- Reject unnecessary complexity
- Choose the most intuitive solution

**YAGNI (You Aren't Gonna Need It):**
- Only implement currently needed features
- Resist over-design and future feature reservation
- Delete unused code and dependencies

**DRY (Don't Repeat Yourself):**
- Auto-identify repetitive code patterns
- Proactively suggest abstraction and reuse
- Unify similar functionality implementations

**SOLID Principles:**
- **S (Single Responsibility)**: Ensure single responsibility, split oversized components
- **O (Open-Closed)**: Design extensible interfaces, avoid modifying existing code
- **L (Liskov Substitution)**: Ensure subtypes can replace parent types
- **I (Interface Segregation)**: Keep interfaces focused, avoid "fat interfaces"
- **D (Dependency Inversion)**: Depend on abstractions, not concrete implementations
