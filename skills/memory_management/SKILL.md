# 记忆管理技能

## 功能概述

帮助用户记录和检索重要信息，建立个人知识库。

## 使用场景

### 创建记忆

当用户说以下话语时，使用 `create_memory` 工具：
- "记住我喜欢喝咖啡"
- "帮我记录一下..."
- "保存这个信息"
- "记一下我的生日是..."

**参数说明：**
- `content`: 记忆内容 (必填)
- `category`: 分类 - personal/work/preference/fact (默认personal)
- `tags`: 标签列表 (可选)
- `importance`: 重要程度 1-5 (默认3)

### 查询记忆

当用户说以下话语时，使用 `search_memories` 工具：
- "我之前说过..."
- "查一下我..."
- "还记得我..."
- "告诉我关于..."

**参数说明：**
- `query`: 查询内容 (必填)
- `limit`: 返回数量 (默认5)

### 更新记忆

当用户说以下话语时，使用 `update_memory` 工具：
- "修改之前的记录"
- "更新一下..."
- "我之前说的不对，应该是..."

### 删除记忆

当用户说以下话语时，使用 `delete_memory` 工具：
- "删除那个记忆"
- "忘掉..."
- "不需要记住..."

**⚠️ 重要：删除操作必须获得用户确认！**

## 记忆分类

| 分类 | 说明 | 示例 |
|------|------|------|
| personal | 个人信息 | 生日、喜好、习惯 |
| work | 工作相关 | 项目信息、同事信息 |
| preference | 偏好设置 | 喜欢的食物、颜色 |
| fact | 事实信息 | 学到的知识、重要数据 |

## 最佳实践

### 内容提炼

记录记忆时，应该：
1. 去除口语化表达
2. 保留核心信息
3. 使用简洁清晰的语句

**示例：**
- 用户: "那个，我想让你记住，就是我喜欢在早上喝咖啡"
- 提炼: "用户喜欢在早上喝咖啡"

### 智能分类

根据内容自动判断分类：
- 包含"喜欢"、"讨厌"、"偏好" → preference
- 包含"生日"、"家庭"、"住址" → personal
- 包含"项目"、"会议"、"同事" → work
- 其他 → fact

### 标签建议

自动提取关键词作为标签：
- "我喜欢喝咖啡" → ["咖啡", "喜好"]
- "我的生日是3月15日" → ["生日", "个人信息"]

## 示例对话

**创建记忆：**
```
用户: 记住我喜欢喝美式咖啡，不加糖
助手: [调用 create_memory]
```

**查询记忆：**
```
用户: 我喜欢喝什么咖啡来着？
助手: [调用 search_memories]
```

**更新记忆：**
```
用户: 我之前说喜欢美式，现在改喝拿铁了
助手: [调用 update_memory]
```
